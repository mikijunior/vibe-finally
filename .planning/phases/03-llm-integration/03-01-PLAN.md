Build the LiteLLM client, structured-output schemas, system prompt, and portfolio-context builder, then wire them into a new `POST /api/chat` endpoint that loads portfolio state, calls the LLM, parses the structured response, persists messages to `chat_messages`, and returns the parsed response. ## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → /api/chat | Untrusted user message text (max 2000 chars); crosses into LLM context. |
| Backend → OpenRouter/Cerebras | Internal prompts contain portfolio state + chat history; outbound API call carries PII (positions, cash) to a third-party. |
| LLM → Backend | Model output is parsed as structured JSON; untrusted text from the LLM flows into DB writes via executor (Plan 03-02). |
| Chat persistence → DB | chat_messages.actions JSON column receives LLM-derived action lists. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-03-01 | Tampering | `POST /api/chat` body | low | mitigate | Pydantic `extra="forbid"` + `message` min_length=1, max_length=2000 rejects malformed/oversized input. Whitespace-only validator catches `"\n"`. |
| T-03-02 | Information Disclosure | LLM prompt outbound | medium | accept | OpenRouter is a trusted subprocessor; documented in `cerebras` skill. CASH/POSITIONS sent intentionally for AI analysis. No mitigation needed beyond trusting the provider. |
| T-03-03 | Tampering | Structured output parse | medium | mitigate | `ChatResponse.model_validate_json(content)` rejects malformed output via Pydantic v2 — raises `LLMError`. Plan 03-02 adds a one-shot retry with corrective system message. |
| T-03-04 | Denial of Service | LLM API latency | low | mitigate | LiteLLM call wrapped in `asyncio.to_thread` so the FastAPI event loop is not blocked. Default `reasoning_effort="low"` for fast Cerebras inference. LLM errors return 503, not 500. |
