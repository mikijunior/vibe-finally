2. In `create_stream_router`, modify the inner `stream_prices` handler to:
   - Return a `StreamingResponse` with `media_type="text/event-stream"` and these headers:
     - `Cache-Control: no-cache`
     - `Connection: keep-alive`
     - `X-Accel-Buffering: no` (already present)
     - `Content-Encoding: identity` (defensive — some proxies default to gzip and break SSE)

3. Update `_generate_events` to:
   - Yield `retry: 1000\n\n` first (browser auto-reconnect after 1s).
   - Yield `: connected\n\n` (an SSE comment line) so the client immediately sees something.
   - Initialize `last_version = -1` and `last_keepalive = time.monotonic()`.
   - Loop:
     - `if await request.is_disconnected(): break` (SSE-05 — graceful disconnect).
     - `current_version = price_cache.version`.
     - `now = time.monotonic()`.
     - If `current_version != last_version`:
       - `last_version = current_version`.
       - `prices = price_cache.get_all()`.
       - If `prices` is non-empty, build payload `json.dumps({ticker: update.to_dict() for ticker, update in prices.items()})` and yield `f"data: {payload}\n\n"`.
     - If `now - last_keepalive >= _KEEPALIVE_INTERVAL_S`:
       - yield `: keepalive\n\n` (SSE-04 — proxy-busting comment, no data).
       - `last_keepalive = now`.
     - `await asyncio.sleep(_CADENCE_S)`.
   - Catch `asyncio.CancelledError` and log at info level (uvicorn cancels on client disconnect, must not raise).

4. Keep the existing log statements (`SSE client connected`, `disconnected`, `cancelled`) and the `_generate_events` signature (price_cache, request, interval) intact — the existing lifespan tests import `create_stream_router` and any signature change breaks them.

6. Import `time` at the top of the file (currently only `asyncio`, `json`, `logging`, `collections.abc` are imported).

Why: SSE-04 (keepalive every 30s) is what keeps nginx / cloudflare / App Runner from killing the connection. SSE-05 (disconnect detection) prevents generator leaks. Version-throttling ensures the wire isn't spammed when prices don't change.

Avoid: Do NOT switch to WebSockets — SSE is the chosen protocol. Do NOT add `await asyncio.sleep(0)` busy-wait — `_CADENCE_S = 0.5` already throttles CPU. <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → `/api/stream/prices` | Untrusted client opens long-lived EventSource; must not crash on malformed clients, must detect disconnects, must throttle send rate |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-SSE-01 | Denial of Service | Many concurrent SSE clients overwhelm generator loop | medium | mitigate | Single asyncio sleep cycle drives all clients per cache version; CPU cost is O(N_clients) per tick, capped by `_CADENCE_S=0.5` and the PriceCache thread-safe lock; under Phase 4's single-user model N_clients=1 |
| T-02-SSE-02 | Information Disclosure | SSE event payload leaks ticker prices | low | accept | Public prices are intentional — same data the watchlist panel renders |
| T-02-SSE-03 | Denial of Service | Slow client stalls the shared generator | medium | mitigate | Per-client generator buffered by asyncio; httpx/starlette handles back-pressure; `X-Accel-Buffering: no` + `Cache-Control: no-cache` tell proxies not to buffer |
| T-02-SSE-04 | Denial of Service | Client never disconnects; generator leaks | high | mitigate | `request.is_disconnected()` is awaited each iteration; `asyncio.CancelledError` caught and logged; the lifespan test in Phase 1 already exercises the disconnect path |
| T-02-SSE-05 | Tampering | Cache version race — two updates emit duplicate events | low | mitigate | `PriceCache._lock` serializes updates; `version` is monotonic; generator reads `version` once per loop and only emits on strict inequality |
