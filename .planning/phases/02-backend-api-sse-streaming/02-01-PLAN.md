`backend/app/api/schemas.py` — Pydantic v2 request/response models. Use `from pydantic import BaseModel, Field, ConfigDict`. Required models:

- `HealthResponse(BaseModel)`: `status: str` (always "ok").
- `TradeRequest(BaseModel)`: `ticker: str = Field(min_length=1, max_length=10, pattern=r"^[A-Za-z]+$")`, `quantity: float = Field(gt=0)`, `side: str = Field(pattern=r"^(buy|sell)$")`. Validator on `model_validator` (or `field_validator` on side) that uppercases `ticker` and rejects empty after strip. (Side is always lowercase per the regex; ticker is normalized to upper.)
- `PositionResponse(BaseModel)`: `ticker`, `quantity: float`, `avg_cost: float`, `current_price: float`, `unrealized_pnl: float`, `pnl_percent: float`. All monetary values are dollars on the wire.
- `PortfolioResponse(BaseModel)`: `cash_balance: float`, `positions: list[PositionResponse]`, `total_value: float`. `total_value = cash_balance + sum(qty * current_price)`.
- `TradeResponse(BaseModel)`: `trade: dict` (the inserted trade row from TradeRepository), `position: dict | None` (the updated position, or null when selling to zero), `cash_balance: float` (post-trade).
- `SnapshotResponse(BaseModel)`: `id`, `total_value: float`, `recorded_at`.
- `PortfolioHistoryResponse(BaseModel)`: `snapshots: list[SnapshotResponse]`.
- `WatchlistAddRequest(BaseModel)`: `ticker: str = Field(min_length=1, max_length=10, pattern=r"^[A-Za-z]+$")`. Validator uppercases ticker.
- `WatchlistEntry(BaseModel)`: `ticker`, `added_at`, `price: float` (from PriceCache, or 0.0 if not yet seen).
- `WatchlistResponse(BaseModel)`: `entries: list[WatchlistEntry]`.
- `WatchlistMutationResponse(BaseModel)`: `ticker`, `action: str` ("added" | "removed"), `already_present: bool` (only meaningful for add; False for remove).

`backend/app/api/portfolio.py` — three endpoints. Use `APIRouter(prefix="/api/portfolio", tags=["portfolio"])`.

- `GET /` (path ""): returns PortfolioResponse. Implementation: load user via `UserRepository.get()` (404 if None), load positions via `PositionRepository.get_all()`, attach `current_price = price_cache.get_price(pos['ticker'])` for each position (default 0.0 if cache miss — never 500), compute `unrealized_pnl = (current_price - avg_cost) * quantity` and `pnl_percent = (current_price - avg_cost) / avg_cost * 100` when `avg_cost > 0` else 0.0, compute `total_value = cash_balance + sum(qty * current_price for each position)`. Return Pydantic model (FastAPI auto-serializes).
- `POST /trade`: body is TradeRequest. Step 1 — load current price from `price_cache.get_price(ticker)`. If `current_price is None`, return HTTPException(400, detail=f"No live price for {ticker}; add to watchlist first or wait for first tick"). Step 2 — if `side == 'buy'`, compute `cost_cents = to_cents(current_price * quantity)`, load user via `UserRepository.get()`, check `cash_balance_cents = to_cents(cash_balance) >= cost_cents`. If not, return 400 with detail=`"Insufficient cash: need ${cost/100:.2f}, have ${cash/100:.2f}"`. Step 3 — if `side == 'sell'`, load position via `PositionRepository.get_one(ticker)`. If missing or `position.quantity < quantity`, return 400. Step 4 — load existing position (if any) and compute the new position state. For buy: `new_qty = existing.quantity + quantity; new_avg_cost = (existing.avg_cost * existing.quantity + current_price * quantity) / new_qty` (always store via to_cents on the result; for the very first buy, existing is None and avg_cost = current_price). For sell: `new_qty = existing.quantity - quantity; new_avg_cost = existing.avg_cost` (unchanged). Step 5 — call `UserRepository.adjust_cash(delta_cents)` where `delta_cents = -cost_cents` for buy, `+to_cents(current_price * quantity)` for sell. Step 6 — `PositionRepository.upsert(ticker, new_qty, new_avg_cost)` if `new_qty > 0`, else `PositionRepository.delete(ticker)`. Step 7 — `TradeRepository.insert(ticker, side, quantity, current_price)` returns the row. Step 8 — `SnapshotRepository.insert(total_value_dollars)` where `total_value_dollars = new_cash_balance_dollars + sum(p.quantity * price_cache.get_price(p.ticker) or 0 for p in all_positions_after_trade)`. Step 9 — load the post-trade position row via `PositionRepository.get_one(ticker)` (or None if deleted) and return TradeResponse with trade dict, position dict (or None), and cash_balance in dollars.
- `GET /history`: returns PortfolioHistoryResponse with all snapshots from `SnapshotRepository.list_all()` ordered by recorded_at ASC. If empty list, still return 200 with `snapshots: []`.

All three handlers MUST be `async def`. Wrap unexpected exceptions in HTTPException(500, "internal error") only after logging; do not swallow validation errors (FastAPI handles 422). Use the dep providers from `app.api.deps` for everything.

`backend/app/api/watchlist.py` — three endpoints. Use `APIRouter(prefix="/api/watchlist", tags=["watchlist"])`.

- `POST /`: body is WatchlistAddRequest. Normalize ticker to upper. Call `WatchlistRepository.add(ticker)` which returns the row (idempotent due to INSERT OR IGNORE — already_present = False only when the row was newly inserted; you can detect this by checking if a row with the same ticker was already there before the call, or simply always call `WatchlistRepository.exists(ticker)` first to detect, then add). To keep it clean: call `exists()` first; if True return 200 with `action="added", already_present=true`. ## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → `/api/portfolio/trade` | Untrusted client sends `{ticker, quantity, side}`; must be validated for ticker format, positive quantity, valid side, sufficient cash/shares |
| Client → `/api/watchlist` POST | Untrusted client sends `{ticker}`; must be validated for uppercase letter format before DB write |
| HTTP → `/api/*` | Untrusted input crosses the API boundary; Pydantic models + explicit validators must reject malformed payloads before any DB mutation |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Tampering | POST /api/portfolio/trade — negative qty, absurd qty, ticker spoofing | high | mitigate | Pydantic `quantity: float = Field(gt=0)`, `side: Literal["buy","sell"]`, ticker regex `^[A-Za-z]+$`; reject oversized qty (e.g. > 1_000_000) with 422; validate cash/shares sufficiency before any DB write |
| T-02-02 | Information Disclosure | GET /api/portfolio — leaks cash balance + positions | low | accept | Single-user demo, no PII, no auth model exists; documented in CLAUDE.md |
| T-02-03 | Denial of Service | POST /api/watchlist spam-adds tickers growing PriceCache unboundedly | low | mitigate | Ticker regex caps length at 10 chars and uppercase letters only; PriceCache size stays <100 in practice |
| T-02-04 | Repudiation | Trade execution with no audit trail | low | mitigate | Every successful trade inserts a row in `trades` AND a row in `portfolio_snapshots` for the post-trade value |
| T-02-05 | Tampering | Concurrent trades racing on the same connection | medium | mitigate | `UserRepository.adjust_cash` uses atomic `UPDATE ... cash_balance = cash_balance + ?` against the single aiosqlite connection; aiosqlite serializes statements on one connection; no torn writes possible within a single process |
| T-02-06 | Tampering | DELETE /api/watchlist/{ticker} with arbitrary path | low | mitigate | FastAPI path param is `ticker: str` uppercased before DB hit; `WatchlistRepository.remove` is no-op when absent and returns False → 404 |
| T-02-SC | Tampering | uv/uvicorn installs | high | mitigate | Already-verified stack (Phase 1 used fastapi 0.115+, aiosqlite 0.22+); only new deps in this phase are pytest ecosystem which is dev-only; no new runtime packages needed |
