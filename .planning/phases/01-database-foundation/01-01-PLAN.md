`backend/app/db/connection.py` — implement the lazy singleton pattern per D-09, D-10, D-06:
- Module-level constants for the DB path. The file lives at `backend/app/db/connection.py`. Use `Path(__file__).resolve().parents[3]` to get the `backend/` dir, then go up one more parent to get project root, then `project_root / "db" / "finally.db"`. Define a small helper `_project_root() -> Path` that returns `Path(__file__).resolve().parents[3].parent` so the intent is explicit. Then `DB_PATH = _project_root() / "db" / "finally.db"`.
- Private module-level state: `_connection: aiosqlite.Connection | None = None` and `_init_lock = asyncio.Lock()` for thread-safe init in async context.
- `async def _enable_wal(db: aiosqlite.Connection) -> None`: executes `PRAGMA journal_mode=WAL` and awaits a fetch of `PRAGMA journal_mode` to confirm WAL is active. Per D-06 and the RESEARCH "Pitfall 1" note, this only needs to run once on first connection because WAL persists in the file.
- `async def _ensure_schema(db: aiosqlite.Connection) -> None`: per D-10, runs `SELECT name FROM sqlite_master WHERE type='table'`, checks for `users_profile`, and if missing reads `schema.sql` (using `Path(__file__).parent / "schema.sql"`) and runs `db.executescript(SCHEMA_SQL)`. Then calls `_seed_defaults(db)` from `seed.py`.
- `async def _seed_defaults(db: aiosqlite.Connection) -> None`: inserts one row into `users_profile` with `id=DEFAULT_USER_ID` and `cash_balance=DEFAULT_CASH_CENTS`, and one row per ticker in `DEFAULT_TICKERS` into `watchlist` with a fresh `uuid.uuid4()` id and `datetime.now(timezone.utc).isoformat()` timestamp. Wraps inserts in a transaction (`await db.commit()` once at the end). Guard against re-seeding by checking if the user row already exists (SELECT then early-return).
- `async def get_db() -> aiosqlite.Connection`: lazy singleton — if `_connection is None`, acquires `_init_lock`, checks again (double-checked locking pattern), opens `aiosqlite.connect(str(DB_PATH))`, sets `db.row_factory = aiosqlite.Row` (per RESEARCH Pitfall 3), calls `_enable_wal(db)`, then `_ensure_schema(db)`, then assigns to `_connection`. Returns the connection.
- `async def init_db() -> aiosqlite.Connection`: public wrapper that just calls and returns `get_db()`. Per D-04 it's part of the public API.
- `async def close_db() -> None`: closes the connection if open, sets `_connection = None`. Per D-04 part of the public API.
