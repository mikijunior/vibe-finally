`backend/app/db/cents.py`:
```python
"""Monetary conversion helpers.

All DB columns store integer cents. API boundaries use float dollars.
"""
from __future__ import annotations


def to_cents(dollars: float) -> int:
    """Convert a dollar float to integer cents. Rounds half-to-even (banker's rounding)."""
    return round(dollars * 100)


def from_cents(cents: int) -> float:
    """Convert integer cents back to a dollar float."""
    return cents / 100.0


def format_dollars(cents: int) -> str:
    """Format cents as a human-readable dollar string, e.g. 123456 -> '$1,234.56'."""
    return f"${from_cents(cents):,.2f}"
```

`backend/app/db/repositories/user.py` — `UserRepository`:
- Class-level constant `USER_ID = "default"` (matches seed.DEFAULT_USER_ID).
- `async def get(self) -> dict`: SELECT id, cash_balance (raw cents), created_at FROM users_profile WHERE id='default'. Returns `{"id": ..., "cash_balance": from_cents(raw_cents), "created_at": ...}`. Returns None if no row.
- `async def update_cash(self, new_cash_dollars: float) -> None`: UPDATE users_profile SET cash_balance = to_cents(new_cash_dollars) WHERE id='default'. Commit.
- `async def adjust_cash(self, delta_cents: int) -> int`: UPDATE users_profile SET cash_balance = cash_balance + delta_cents WHERE id='default', then SELECT cash_balance, return new raw cents. Useful for atomic +/- without round-trip through dollars. Commit.

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-03 | Tampering | Repository SQL strings | high | mitigate | All SQL uses parameterized `?` placeholders; ticker/side/role values flow only as parameters, never interpolated |
| T-01-04 | Tampering | Ticker symbol injection | medium | mitigate | `WatchlistRepository.add` normalizes ticker to uppercase before storage; future API layer should also enforce ticker format (regex) |
| T-01-05 | Information Disclosure | cents helpers leak precision | low | accept | `format_dollars` and `from_cents` are public; precision is exactly 2 decimal places by design |
| T-01-06 | Repudiation | Trade/chat insert with no audit | low | accept | `executed_at` and `created_at` are auto-populated by SQLite `datetime('now')`; sufficient for v1 single-user |
